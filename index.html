HTML

  • The head element contains information about the webpage.
  • The title element defines the title for a webpage
  • Metadata is typically not seen by users, but it contains important information about accessibility, search engines, and performance.
  • The body element represents the visible content shown to the user.

CSS

  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet
  • Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only, e.g. <h1 style="color:red;">Prework Study Guide</h1>
  • The internal style sheet contains CSS rules for the webpage in the head section of the HTML file using <style> element
  • When using an external style sheet, we link the file using a <link> element placed in the HTML's <head>, e.g. <link rel="stylesheet" href="./assets/style.css" />
  • The period preceding [.card e.g.] designates this as a class selector. Any HTML element assigned to the class [.card], will gain these property assignments from the CSS
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status //checks what branch we are currently on
  • git checkout -b feature/{branch-name} //creates a new branch and switches to it
  • git add -A //add all changes in the working directory to the staging area
  • git add file_name_1 file_name_2 ... //add specified files to the staging area
  • git commit -m "add starter code" //write message as imperative commands in the present tense.
  • git pull origin main //fetch and download content from a remote repository and immediately update the local repository to match that content
  • gt push origin feature/{branch-name} //push code to a branch called feature/{branch-name} in this specific remote repository called "origin"

JavaScript

  • file is linked within the <body> element.
  • var keyword is used to declare variables. After the var keyword, we give the variable a name.
